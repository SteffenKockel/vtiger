5. Known Issues

We have fixed many more issues in the present release. However at the end-of-the-day we have some known issues left open, which will be addressed in the subsequent releases.

Few Known issues:

  • Field length validation is not available.
  • System details information will not be complete, if operating system applies restriction.
  • Created Time and Modified Time depends on the timezone format set in php configuration or server.
  • Pagination is not supported in Global Search.

For a complete list of known issues, see: vtiger Bug Tracker

6. Limitations

  • Limited Field Level Security in Inventory PDF and Dashboards.
  • Chat does not work during the heavy load in the server.
  • In future release, week view UI will be changed, so all the events are displayed at start time of the event.
  • If events start time is lesser than the Calendar start time, it will not be displayed in the calendar view.

7.Upgrading Procedure

This section is only for the existing vtiger CRM users. If you are planning to use vtiger CRM first time, please skip this section.

Important Note:

  • We recommend users to take a backup of the existing database and source code before migrating to vtiger CRM 5.0.4.
  • If you have customized the source code and database, please contact your developers or write to us at info@vtiger.com so that we can suggest a proper migration path.

Step 1: Copy vtiger CRM 5.0.4 - Patch files in to your vtiger CRM root directory.

Step 2: Migrate your database from earlier version to the latest version.

To upgrade from v4.2.3, 5.0.0, 5.0.1, 5.0.2, 5.0.3-rc2, 5.0.3, 5.0.4-rc to v5.0.4

  1. Select Settings --> Other Settings --> Migration Wizard
  2. Make the recommended changes if any for your install before proceeding, if you need utf-8 support.
  3. In the Migration Wizard, select the options according to your requirements
  4. It will take some time to migrate the data. Kindly do not close the browser during migration process.
